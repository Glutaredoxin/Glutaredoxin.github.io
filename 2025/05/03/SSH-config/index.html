远程连接时可能出现的问题

出现WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED！

解决：
ssh-keygen -R 192.168.XX.XXX

下载驱动

sudo apt-get install g++

sudo apt-get install gcc

sudo apt-get install cmake

sudo apt-get remove --purge nvidia*

sudo nano /etc/modprobe.d/blacklist.conf

加入

blacklist nouveau
options nouveau modeset=0

sudo update-initramfs -u

sudo reboot

lsmod | grep nouveau

无输出证明正确

sudo service gdm3 stop

下载推荐驱动（后面带有recommanded）：
sudo apt-get install nvidia-driver-XXX

sudo reboot

使用nvidia-smi查看是否成功

安装CUDA

直接使用CUDA Toolkit的命令行安装

然后添加

export PATH=/usr/local/cuda-11.3/bin${PATH:+:${PATH}}
export LD_LIBRARY_PATH=/usr/local/cuda-11.3/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}

其中11.3需要改成自己下的版本，这里我们使用的是12.4.

利用nvcc -V查看是否安装成功。

安装docker

——用国内镜像源安装docker——

国内源添加密钥

curl -fsSL https://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo apt-key add -

国内源添加软件仓库

sudo add-apt-repository "deb [arch=amd64] https://mirrors.aliyun.com/docker-ce/linux/ubuntu focal stable"

安装docker-ce

sudo apt install -y docker-ce

——用国内镜像源安装nvidia docker工具包——

修改docker的工作目录以及国内镜像，打开配置文件(如果没有文件会自动创建)，添加以下配置

vi /etc/docker/daemon.json

重启服务

配置用户组，使得brl可以直接使用docker，而不是sudo docker

导入镜像

存储的镜像和docker是在“/media/brl/d177acba-c04e-4a5c-b4d2-f7005e8c4035”中

docker import XXX.tar XX

其中，XX表示为这个镜像命名名称。

docker images # 查看镜像

docker run --gpus 2 -dit -p2345:22 --name=[container名称] -h=LAB_VM --ipc=host -v /home/disk1/stu_data:/data [镜像名] bin/bash # 查看镜像

2、docker container ls -a

3、docker container start xxxx

4、docker ps

5、docker exec -it [your docker] /bin/bash

6、service ssh restart

将文件全部挂在到/home/disk1/stu_data

lsblk # 查看文件/media/brl/d177acba-c04e-4a5c-b4d2-f7005e8c4035在那个dev中

进行挂载

sudo mkdir -p /home/disk1/stu_data # 创建一个文件夹用于挂载

sudo mount /dev/sda /home/disk1/stu_data # 进行挂载
